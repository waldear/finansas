FINANSAS / FINFLOW - INSTRUCCIONES RAPIDAS
=========================================

Este proyecto usa Next.js + Supabase + Gemini.

1) Requisitos
- Node.js 20+
- Proyecto en Supabase (Auth + Postgres + Storage)
- API key de Gemini (Google AI Studio)

2) Instalar y correr (local)
```bash
npm install
npm run dev
```
Abrir: http://localhost:3000

3) Variables de entorno (NO commitear keys)
Crear `.env.local` (ver `.env.local.example`) y completar:

- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- GEMINI_API_KEY
- NEXT_PUBLIC_SITE_URL (ej: http://localhost:3000)

Opcional:
- GEMINI_MODEL (recomendado: gemini-2.5-flash)
- ASSISTANT_FREE_MONTHLY_REQUEST_LIMIT (default 40)
- ASSISTANT_PRO_MONTHLY_REQUEST_LIMIT (default 1200)

4) SQL en Supabase (RLS + tablas)
En Supabase -> SQL Editor, ejecutar en este orden:
1. supabase-schema.sql
2. supabase-copilot.sql
3. supabase-advanced.sql
4. supabase-spaces.sql

5) Deploy (Vercel)
Ver DEPLOY-VERCEL.md o README.md.

Seguridad
- GEMINI_API_KEY y STRIPE_SECRET_KEY son server-only. Nunca deben ir en el cliente ni en el repo.
- Si una key se filtro por chat/Git, rotala y actualizala en Vercel/Supabase.
